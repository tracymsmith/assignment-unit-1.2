# 1 - What is the command for listing the files in the current folder (directory)?

'ls'

# 2 - What is the command for changing directories?

'cd' plus name of directory

# 3 - What is the command for going "up" a folder (to a parent directory)?

'cd ..' OR up arrow on keyboard

# 4 - What is a quick way to create a new folder?

'mkdir' plus name of folder

# 5 - What output would you expect when you run the command 'ls' in an empty folder? What about a folder containing files?

'ls' command for an empty folder would result in blank output; 'ls' command for a
folder containing files output would list all the files in the folder

# 6 - What is a quick way to create a new file?

'touch' plus name of file with appropriate extension i.e., file.js or doc.docx or spreadsheet.xls or index.html, etc.

# 7 - Run 'git --version' in your terminal. What is the output?

'git version 2.39.3 (Apple Git-146)'

# 8 - Why would you run the command 'git status'?

To determine if there were any files (red in color) in the staging area that need to be (haven't been) committed and/or to verify that
the files you want to send to repository on personal GitHub have been committed (green in color).

# 9 - What does the command 'git add .' do?

It changes uncommitted files (red in color) in the staging area to committed files (green in color) that are
now ready to be pushed to repository on personal GitHub.

# 10 - What is the command to push your committed changes to your repo on GitHub?

After 1st running 'git status' command to confirm the files have been committed (green in color), then
run command 'git push origin main' to push files to repository on personal GitHub.
